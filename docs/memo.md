## 通知サービス実装メモ

### 目的

弁当注文自動化の実行結果を通知する。

現時点では、通知機能は初回リリース対象外とする。
まずは本番運用を開始し、会社携帯への購入通知とログ・スクリーンショットによる目視確認で運用する。

通知機能は、運用開始後の追加リリースで実装する。

## 実装方針

通知は以下の2種類をまとめて実装する。

* LINE通知
* メール通知

優先度は以下。

1. LINE通知
2. メール通知

## LINE通知の方針

LINEは普段確認しやすいため、運用通知に向いている。

使用候補は LINE Messaging API。
LINE Notify は終了済みのため、新規実装では使わない。

### LINE通知で送る内容

#### 注文完了

```text
【弁当注文】注文が完了しました

対象日: 2026-06-29
商品: A弁当 ライス大
金額: 600円
```

#### 既に注文済み

```text
【弁当注文】既に注文済みでした

対象日: 2026-06-29
```

#### 注文失敗

```text
【弁当注文】注文処理に失敗しました

対象日: 2026-06-29
理由: xxx
スクリーンショット: screenshots/xxx.png
```

## メール通知の方針

メール通知は、SMTP実装の学習と障害通知の汎用性を目的に追加する。

Gmail SMTP または SendGrid を候補にする。

### メール通知で送る内容

LINEと同じ内容を送る。

件名例:

```text
【弁当注文】注文完了
【弁当注文】既に注文済み
【弁当注文】注文失敗
```

本文例:

```text
弁当注文自動化の実行結果です。

ステータス: ORDER_CONFIRMED
対象日: 2026-06-29
商品: A弁当 ライス大
金額: 600円
スクリーンショット: screenshots/order-confirmed-xxxx.png
```

## 通知対象

通知するイベントは以下。

| イベント    | LINE通知 | メール通知 | 備考           |
| ------- | -----: | ----: | ------------ |
| 注文完了    |      ○ |     ○ | 本番注文成功       |
| 既に注文済み  |      ○ |     ○ | 二重注文防止が効いた状態 |
| 注文失敗    |      ○ |     ○ | 最重要          |
| 土日スキップ  |      × |     × | 通知ノイズになるため不要 |
| 祝日スキップ  |      × |     × | 通知ノイズになるため不要 |
| ドライラン成功 |     任意 |    任意 | 初期は不要        |

## 環境変数案

`.env.example` に追加予定。

```env
# LINE通知
LINE_CHANNEL_ACCESS_TOKEN=
LINE_USER_ID=

# メール通知
MAIL_FROM=
MAIL_TO=
SMTP_HOST=
SMTP_PORT=587
SMTP_USER=
SMTP_PASSWORD=
```

既にメール用の環境変数が存在する場合は、それを流用する。

## 実装予定ファイル

```text
src/services/notificationService.ts
```

責務:

* LINE通知を送る
* メール通知を送る
* 注文結果に応じた通知文面を生成する
* 通知失敗時に本体処理を壊さない

## 実装イメージ

```ts
type NotifyInput = {
  status: string;
  targetDate?: string;
  menuName?: string;
  price?: number;
  reason?: string;
  screenshotPath?: string;
};

export async function notifyOrderResult(input: NotifyInput): Promise<void> {
  // LINE通知
  // メール通知
}
```

## 注意点

* 通知にログインID・パスワード・カード情報を含めない
* 画面全文を通知しない
* スクリーンショットのファイルパスだけ通知する
* 通知失敗で注文処理自体を失敗扱いにしない
* Cloud Run環境では、ローカルの screenshots パスをそのまま見られない可能性がある
* Cloud Storage保存を入れた後は、スクリーンショットURLまたは保存パスを通知する

## リリース方針

### 初回リリース

通知機能なしで本番運用を開始する。

確認方法:

* 実行ログ
* スクリーンショット
* 会社携帯への購入通知
* おべんとね！っと画面での注文済み確認

### 追加リリース1

LINE通知を追加する。

対象:

* 注文完了
* 既に注文済み
* 注文失敗

### 追加リリース2

メール通知を追加する。

対象:

* 注文完了
* 既に注文済み
* 注文失敗

## 現時点の判断

通知機能は後回しでよい。

理由:

* まずは本番運用開始を優先したい
* 会社携帯にも購入通知が来る
* ログとスクリーンショットで確認可能
* 通知機能はLINEとメールをまとめて追加した方が設計しやすい
